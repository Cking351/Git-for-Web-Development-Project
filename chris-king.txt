    37. Walk me through Lambda's git flow. 
    - You start by forking the project on github. You then add your TL as a collaborator and
      clone the project from your forked version. In terminal, you use git clone to create a 
      local directory. You use git checkout -b 'firstname-lastname' to create a branch to work on.
      As you work, you use git add to add the changes you make, git commit every 20 minutes to save
      your work, and git push -u origin branch-name when you are ready to make the change.
      You then create a pull request on github from your specific branch and add your TL
      as a reviewer.
